Important Notes

  • 4-bit mode uses only D4–D7, leaving D0–D3 unconnected. Data is sent as two 4-bit nibbles (high nibble first).
  • RW pin tied to GND keeps the LCD in write-only mode — this is standard practice since reading the busy flag isn't needed (use software delays instead).
  • 10KΩ potentiometer: Connect one end to +5V, the other to GND, and the wiper (middle pin) to LCD pin 3 (V0). Rotate to adjust character contrast.
  • 220Ω resistor on the backlight LED+ limits current to ~15mA, protecting the backlight. Some modules have a built-in resistor — check your datasheet.
  • Voltage levels: RPi4 GPIO outputs 3.3V logic. The HD44780 accepts 3.3V as logic HIGH when powered at 5V, so no level shifter is needed.
  • Software: Use Python with RPi.GPIO or the RPLCD library. For kernel-level, write a character device driver using the GPIO descriptor API (gpiod) — the same approach as your RPi4 GPIO driver work.
  • Power: The LCD draws ~1–2mA (logic) + ~15mA (backlight). The RPi4 5V pin can supply this without issues.
